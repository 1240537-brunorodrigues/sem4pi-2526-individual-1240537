# US106 - Implement Function-Specific Threads in the Parent Process

## User Story

As a PO, I want the simulation controller parent process to have at least two dedicated threads, one for safety violation detection and one for report generation, so that each functionality operates concurrently and independently.

The parent process must create a safety violation detection thread responsible for scanning shared memory for aircraft flight conflicts. A report generation thread must also be created to compile simulation results and respond to safety violation events. Additional threads may be created for other required functionalities. Threads must be managed using mutexes and condition variables for internal synchronization.
