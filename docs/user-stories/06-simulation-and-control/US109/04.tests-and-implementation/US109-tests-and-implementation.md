# US109 - Generate Simulation Report

## 4. Tests and Implementation

### 4.1. Implementation Notes

This user story should implement simulation report generation.

The implementation should include:

* authorization validation for the Flight Control Operator;
* lookup or validation of the selected simulation;
* safe reading of simulation data;
* report generation by the report generation thread;
* inclusion of simulation configuration;
* inclusion of included flights and process statuses;
* inclusion of position history or movement summaries;
* inclusion of safety violation events;
* inclusion of warnings and errors;
* inclusion of final simulation outcome;
* formatting or parsing of the report output;
* meaningful error reporting if report generation fails.

The report must use consistent simulation data and should avoid partially completed time-step information.

### 4.4. Unit Tests

The following unit tests should be implemented:

* Verify that an authorized Flight Control Operator can request a simulation report.
* Verify that an unauthorized user cannot generate a simulation report.
* Verify that a report cannot be generated for a non-existing simulation.
* Verify that simulation configuration is included in the report.
* Verify that included flights are included in the report.
* Verify that flight process statuses are included in the report.
* Verify that aircraft movement or position summaries are included in the report.
* Verify that safety violation events are included in the report.
* Verify that safety violation count is included in the report.
* Verify that warnings and errors are included in the report.
* Verify that final simulation outcome is included in the report.
* Verify that partial reports identify the simulation as still running.
* Verify that report generation failure returns an error.

### 4.5. C Component Tests

The following C-level tests should be implemented:

* Verify that the report generation thread can read shared simulation configuration.
* Verify that the report generation thread can read flight process statuses.
* Verify that the report generation thread can read position history.
* Verify that the report generation thread can read safety violation events.
* Verify that shared data is read under the required synchronization mechanism.
* Verify that the report builder creates a complete report structure.
* Verify that the report formatter creates the expected output.
* Verify that report generation handles missing optional data.
* Verify that report generation handles empty safety violation lists.
* Verify that report generation does not include inconsistent incomplete time-step data.

---

### 4.6. Integration Tests

The following integration tests should be implemented:

* Verify that a completed simulation produces a final report.
* Verify that a running simulation produces a partial/snapshot report.
* Verify that safety violations detected during simulation appear in the report.
* Verify that position data generated by flight processes appears in movement summaries.
* Verify that warnings and errors logged during simulation appear in the report.
* Verify that Java/application layer can request and parse a C-generated report.
* Verify that report generation remains safe while simulation data is being updated.

### 4.7. Acceptance Tests

**Test 1:** Generate report for completed simulation

* **Given** an authenticated Flight Control Operator
* **And** a completed simulation exists
* **When** the operator requests the simulation report
* **Then** the system should generate a report
* **And** the report should include the final simulation outcome

**Test 2:** Report includes simulation configuration

* **Given** a simulation report is generated
* **When** the report is displayed
* **Then** it should include the simulation configuration used for that execution

**Test 3:** Report includes flight summaries

* **Given** a simulation with included flights
* **When** the report is generated
* **Then** it should include a summary of the included flights and their process statuses

**Test 4:** Report includes position information

* **Given** a simulation with recorded aircraft positions
* **When** the report is generated
* **Then** it should include aircraft movement or position summaries

**Test 5:** Report includes safety violations

* **Given** a simulation where safety violations occurred
* **When** the report is generated
* **Then** the report should include the safety violation events
* **And** the total number of safety violations

**Test 6:** Partial report for running simulation

* **Given** a simulation is still running
* **When** the operator requests a report
* **Then** the system should generate a current snapshot report
* **And** the report should indicate that the simulation is still running

**Test 7:** Report generation failure

* **Given** report data cannot be read or compiled
* **When** the operator requests the report
* **Then** the system should display a meaningful report generation error

**Test 8:** Unauthorized operation

* **Given** an authenticated user without permission to generate simulation reports
* **When** the user requests a simulation report
* **Then** the system should deny access
